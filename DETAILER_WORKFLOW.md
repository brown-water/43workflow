# Master Planner Architecture & Detailer Workflow Summary

## Strategic Shift
The Master Planner is evolving from a manual Excel/VBA tool into a "cinematic" HTML-based application. It will act as the **Intent of Record**, running locally (Zero-Install Vanilla JS) and eventually migrating to SharePoint (React/SPFx). It functions as a "diff engine," comparing the user's mutable "Intent" against the rigid "Legal Status" provided by OAIS. Discrepancies are surfaced as actionable "Sync Required" alerts.

## Data Pipeline (The Read Bridge)
The application will consume a clean `state.csv` generated by a Power Automate/Power Query pipeline. This pipeline ingests daily `%`-delimited flat files (e.g., `SUPPLY_sYNTH.txt`) from the MECH FTP server and drops the processed CSV into a SharePoint Watch Folder. We have mapped the ODIS data dictionary (`headers.csv` and `SLATE_Fieldlisting.pdf`) to accurately parse fields like `ICOMM.SCRN.RSLT` (CSR Codes).

## Board Eligibility Guardrail
To avoid the fragility of automated PDF parsing, board eligibility rules (e.g., Year Groups, AQDs) will be maintained via an Expert-in-the-Loop process. The rules will be transcribed annually into a `board_rules.csv` or config screen, allowing the HTML app to automatically cross-reference against the officer dataset with 100% accuracy.

## The "Intent of Record" Schema (Detailer Focus)
We designed a modern JSON schema from scratch, abandoning legacy mindsets. This schema represents the **Detailer's Workflow**, focusing on how officers are slated to future commands:

1.  **JO Slate:** Tracks `Target_Ultimate_UIC`, `Target_PRD`, `Slate_Cycle`, and `FITREP_Expectation`. It differentiates between an "Incomplete" slate intent and a "Complete" legally binding OAIS order.
2.  **Pop-up Vacancies & Senior Slates:** Introduces a `Billets_Priority` array to track unfilled seats independently of officers, ensuring they are queued for the next DH or CO slate.
3.  **Board Shadows & Training Pipelines (I-Stops):** Tracks complex, multi-stop training tracks (especially for COs, which take 7-9 months). The schema includes an `I_Stops` array and an `Order_Trigger_Date` (calculated ~8 months prior to the report date) to auto-populate a "Needs Orders Drafted" queue.
4.  **BNRs, NOMs, and Advanced Quals:** Handles disruptions to the normal slate. It tracks `Nomination_Status` for VIP packages (e.g., CNO Speech Writer) and allows for a `Manual_Override_Flag` when a JO is selected for TOPGUN or SFTI, immediately cuing the UI that OAIS sync is required.

## Next Steps
This summary strictly defines the **Detailer's** portion of the workflow. The immediate next phase of the project is to drill down into the **Placement Officer's** portion of the workflow to ensure the Master Planner comprehensively supports both sides of the detailing and placement process.
